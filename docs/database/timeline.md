# Таблиця `timeline`

## Призначення

Таблиця `timeline` є журналом життя спільноти.

Вона містить усі важливі події, що відбуваються із користувачами та системою.

Timeline використовується для:

- відображення історії активності;
- побудови хронології;
- аналітики;
- автоматизації;
- формування стрічки подій.

Timeline не є журналом змін даних.

Для цього використовується таблиця `audit_log`.

---

# Відповідальність

Таблиця відповідає лише за реєстрацію подій.

Вона не дублює дані інших таблиць.

---

# Поля

| Поле | Тип | NULL | Опис |
|------|-----|------|------|
| id | BIGSERIAL | ❌ | Первинний ключ |
| user_id | BIGINT | ✅ | Користувач, до якого відноситься подія |
| event_type | VARCHAR(50) | ❌ | Тип події |
| entity_type | VARCHAR(50) | ❌ | Тип сутності |
| entity_id | BIGINT | ✅ | Ідентифікатор сутності |
| source | VARCHAR(50) | ❌ | Джерело події |
| title | VARCHAR(255) | ❌ | Короткий заголовок |
| description | TEXT | ✅ | Детальний опис |
| metadata | JSONB | ✅ | Додаткові дані |
| created_at | TIMESTAMP | ❌ | Дата події |

---

# Первинний ключ

```
id
```

---

# Зовнішні ключі

```
user_id → users.id
```

# Типи подій

Приклади:

- user_joined
- user_left
- first_message
- profile_updated
- message_saved
- event_registered
- event_attended
- recommendation_received
- ai_profile_updated
- import_completed

Перелік типів визначається через ENUM.

Перелік визначається через ENUM.

Створюються індекси:

---

# Правила оновлення

Записи Timeline не редагуються.

Допускається лише створення нового запису.

---

# Видалення

Записи Timeline не видаляються.

---

# Причини такої структури

Timeline є універсальною таблицею подій.

Вона дозволяє відобразити історію розвитку користувача та спільноти незалежно від конкретної сутності.

---

# Майбутній розвиток

Можливе додавання:

- важливості події;
- кольору відображення;
- іконки;
- зв'язку між подіями.
