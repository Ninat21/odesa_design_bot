# Таблиця `event_feedback`

## Призначення

Таблиця `event_feedback` містить відгуки учасників про проведені заходи.

Відгук може бути залишений вручну користувачем або імпортований із зовнішнього сервісу.

---

# Відповідальність

Таблиця відповідає за:

- оцінку заходу;
- текстовий відгук;
- рекомендацію заходу;
- джерело отримання відгуку.

---

# Поля

| Поле | Тип | NULL | Опис |
|------|-----|------|------|
| id | BIGSERIAL | ❌ | Первинний ключ |
| event_id | BIGINT | ❌ | Посилання на `events.id` |
| user_id | BIGINT | ❌ | Посилання на `users.id` |
| rating | SMALLINT | ✅ | Оцінка (1–5) |
| review | TEXT | ✅ | Текстовий відгук |
| would_recommend | BOOLEAN | ✅ | Чи рекомендує захід |
| source | VARCHAR(30) | ❌ | Джерело відгуку |
| created_at | TIMESTAMP | ❌ | Дата створення |
| updated_at | TIMESTAMP | ❌ | Дата оновлення |

---

# Первинний ключ

```
id
```

---

# Зовнішні ключі

```
event_id → events.id

user_id → users.id
```

---

# Обмеження

Один користувач може залишити лише один відгук на один захід.

```
UNIQUE(event_id, user_id)
```

# Правила оновлення

Користувач може редагувати свій відгук.

Усі зміни повинні фіксуватися в `audit_log`.

---

# Причини такої структури

Відгуки є окремою сутністю.

Вони не повинні зберігатися в таблиці `events`, оскільки одна подія може мати необмежену кількість відгуків.

---

# Майбутній розвиток

Можливе додавання:

- анонімного режиму;
- оцінки окремих аспектів заходу;
- фотографій;
- реакції організаторів.
