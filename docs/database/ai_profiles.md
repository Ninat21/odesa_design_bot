# Таблиця `ai_profiles`

## Призначення

Таблиця `ai_profiles` містить результати аналізу користувачів, виконаного моделями штучного інтелекту.

Вона не містить фактичних даних користувача.

Уся інформація в таблиці є інтерпретацією AI.

---

# Відповідальність

Таблиця відповідає за:

- узагальнений опис користувача;
- професійний профіль;
- сильні сторони;
- потенційні напрямки розвитку;
- короткі AI-висновки.

---

# Поля

| Поле | Тип | NULL | Опис |
|------|-----|------|------|
| id | BIGSERIAL | ❌ | Первинний ключ |
| user_id | BIGINT | ❌ | Посилання на `users.id` |
| summary | TEXT | ✅ | Короткий опис користувача |
| personality | TEXT | ✅ | Аналіз стилю спілкування |
| strengths | TEXT | ✅ | Сильні сторони |
| growth_areas | TEXT | ✅ | Зони розвитку |
| communication_style | TEXT | ✅ | Стиль комунікації |
| activity_summary | TEXT | ✅ | Узагальнення активності |
| model | VARCHAR(100) | ❌ | AI-модель |
| confidence | NUMERIC(5,2) | ✅ | Загальна впевненість |
| last_analyzed_at | TIMESTAMP | ✅ | Останній аналіз |
| created_at | TIMESTAMP | ❌ | Створення |
| updated_at | TIMESTAMP | ❌ | Оновлення |

---

# Первинний ключ

```
id
```

---

# Зовнішні ключі

```
user_id → users.id
```

---

# Обмеження

```
UNIQUE(user_id)
```

```
user_id

```

# Правила оновлення

Профіль може повністю перегенеровуватися після повторного аналізу.

Попередні версії зберігаються в таблиці `ai_profile_history`.

---

# Джерела

- OpenAI
- Anthropic
- Google Gemini
- локальні моделі

---

# Причини такої структури

AI-профіль існує окремо від фактичного профілю користувача.

Це дозволяє змінювати або повторно генерувати результати аналізу без впливу на персональні дані.

---

# Майбутній розвиток

Можливе додавання:

- кількох моделей одночасно;
- рейтингу достовірності окремих висновків;
- спеціалізованих профілів для різних задач.
