# Таблиця `saved_messages`

## Призначення

Таблиця містить повідомлення Telegram, які були позначені як важливі або корисні.

Вона використовується як проміжний етап між історією повідомлень та базою знань.

Повідомлення можуть додаватися користувачами, адміністраторами або AI.

---

# Відповідальність

Таблиця відповідає за:

- збереження важливих повідомлень;
- класифікацію повідомлень;
- контроль статусу опрацювання;
- фіксацію автора збереження.

---

# Поля

| Поле | Тип | NULL | Опис |
|------|-----|------|------|
| id | BIGSERIAL | ❌ | Первинний ключ |
| message_id | BIGINT | ❌ | messages.id |
| saved_by | BIGINT | ✅ | users.id або NULL, якщо AI |
| bookmark_type | VARCHAR(50) | ❌ | Тип збереження |
| note | TEXT | ✅ | Коментар користувача або адміністратора |
| source | VARCHAR(30) | ❌ | user / admin / ai |
| status | VARCHAR(30) | ❌ | new / reviewed / archived / exported |
| created_at | TIMESTAMP | ❌ | Дата створення |
| updated_at | TIMESTAMP | ❌ | Дата оновлення |

---

# Первинний ключ

```
id
```

---

# Зовнішні ключі

```
message_id → messages.id

saved_by → users.id
```

---

# Обмеження

Одне повідомлення може бути збережене декілька разів різними людьми.

```
UNIQUE(message_id, saved_by, bookmark_type)
```

# Правила оновлення

Після створення запису можна змінювати:

- статус;
- тип;
- примітку.

Посилання на повідомлення змінювати не дозволяється.

---

# Причини такої структури

Таблиця не залежить від Knowledge Base.

Вона дозволяє накопичувати цінні повідомлення, які можуть бути використані в майбутньому для:

- створення статей;
- формування FAQ;
- побудови добірок;
- навчання AI;
- створення дайджестів;
- пошуку найкращих відповідей.
