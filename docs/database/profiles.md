# Таблиця `profiles`

## Призначення

Таблиця `profiles` містить персональну інформацію про користувача, яка не належить до технічних даних Telegram.

Профіль заповнюється самим користувачем, адміністратором або імпортується із зовнішніх джерел за згодою користувача.

---

# Відповідальність

Таблиця відповідає за:

- контактну інформацію;
- професійну інформацію;
- біографію;
- особисті посилання;
- налаштування приватності;
- згоди користувача.

---

| city | VARCHAR(255) | ✅ | Місто |
| country | VARCHAR(255) | ✅ | Країна |

| company | VARCHAR(255) | ✅ | Компанія |
| position | VARCHAR(255) | ✅ | Посада |
| bio | TEXT | ✅ | Біографія |
| instagram | VARCHAR(255) | ✅ | Instagram |
| behance | VARCHAR(255) | ✅ | Behance |
| linkedin | VARCHAR(255) | ✅ | LinkedIn |

---

# Первинний ключ

```
id
```

---

# Зовнішні ключі

```
user_id → users.id
```

---

# Обмеження

Один користувач може мати лише один профіль.

```
UNIQUE(user_id)
```

---

# Індекси

Створюються індекси:

```
user_id
email
city
profession
company
```

---

# Зв'язки

## Один до одного

```
User
```

---

# Правила оновлення

Профіль може змінюватися користувачем.

Кожна зміна записується в Audit Log.

---

# Джерела даних

Допустимі джерела:

- Telegram Bot
- Luma
- Website
- Administrator
- Import

AI не має права змінювати дані профілю.

---

# Конфіденційність

Email зберігається лише після отримання згоди користувача.

Дата народження є персональними даними.

Контактна інформація не повинна використовуватися без відповідної згоди користувача.

---

# Причини такої структури

Профіль винесений в окрему таблицю, щоб не перевантажувати таблицю `users`.

Технічні дані Telegram і персональна інформація мають різний життєвий цикл і різні правила оновлення.

---

# Майбутній розвиток

У майбутньому можливе додавання:

- часового поясу;
- мов, якими володіє користувач;
- досвіду роботи;
- освіти;
- контактних способів зв'язку.
