# Таблиця `notifications`

## Призначення

Таблиця `notifications` містить інформацію про повідомлення, які система надсилає користувачам.

Вона використовується для контролю доставки повідомлень та ведення історії сповіщень.

---

# Відповідальність

Таблиця відповідає за:

- створення повідомлень;
- статус доставки;
- журнал відправлень;
- повторні спроби відправки.

---

# Поля

| Поле | Тип | NULL | Опис |
|------|------|------|------|
| id | BIGSERIAL | ❌ | Первинний ключ |
| user_id | BIGINT | ❌ | users.id |
| notification_type | VARCHAR(50) | ❌ | Тип повідомлення |
| title | VARCHAR(255) | ✅ | Заголовок |
| message | TEXT | ❌ | Текст повідомлення |
| status | VARCHAR(30) | ❌ | Статус |
| sent_at | TIMESTAMP | ✅ | Час відправлення |
| read_at | TIMESTAMP | ✅ | Час прочитання |
| created_at | TIMESTAMP | ❌ | Створення |
| updated_at | TIMESTAMP | ❌ | Оновлення |

---

# Статуси

- pending
- sent
- delivered
- failed
- read

---

# Зовнішні ключі

```
user_id → users.id
```

---

# Індекси

```
user_id

status

created_at
```

---

# Причини такої структури

Дозволяє контролювати історію всіх повідомлень, які надсилає система.
