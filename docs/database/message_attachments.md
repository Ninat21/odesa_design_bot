# Таблиця `message_attachments`

## Призначення

Таблиця містить усі вкладення повідомлень Telegram.

Одне повідомлення може містити необмежену кількість вкладень.

Вкладення не зберігаються безпосередньо в таблиці `messages`.

---

# Відповідальність

Таблиця відповідає за:

- тип вкладення;
- Telegram File ID;
- Telegram Unique File ID;
- назву файлу;
- MIME-тип;
- розмір;
- тривалість;
- роздільну здатність;
- службову інформацію.

---

# Поля

| Поле | Тип | NULL | Опис |
|------|-----|------|------|
| id | BIGSERIAL | ❌ | Первинний ключ |
| message_id | BIGINT | ❌ | messages.id |
| attachment_type | VARCHAR(30) | ❌ | Тип вкладення |
| telegram_file_id | TEXT | ❌ | Telegram File ID |
| telegram_unique_file_id | TEXT | ✅ | Унікальний File ID |
| file_name | VARCHAR(255) | ✅ | Назва файлу |
| mime_type | VARCHAR(100) | ✅ | MIME-тип |
| file_size | BIGINT | ✅ | Розмір у байтах |
| width | INTEGER | ✅ | Ширина |
| height | INTEGER | ✅ | Висота |
| duration | INTEGER | ✅ | Тривалість у секундах |
| thumbnail_file_id | TEXT | ✅ | Telegram File ID прев'ю |
| created_at | TIMESTAMP | ❌ | Створення запису |
| updated_at | TIMESTAMP | ❌ | Останнє оновлення |

---

```
id
```

# Типи вкладень

Підтримуються:

- photo
- video
- animation
- audio
- voice
- video_note
- sticker
- document

Архітектура повинна дозволяти додавання нових типів без зміни структури таблиці.

# Правила оновлення

Після створення вкладення змінювати його характеристики не допускається.

Може оновлюватися лише службова інформація.

# Джерела даних

Допустимі джерела:

- Telethon
- Import

---

# Причини такої структури

Telegram постійно розширює перелік типів вкладень.

Винесення вкладень в окрему таблицю дозволяє:

- не змінювати структуру `messages`;
- підтримувати необмежену кількість вкладень;
- не дублювати поля;
- зменшити розмір основної таблиці повідомлень.
