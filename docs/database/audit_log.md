# Таблиця `audit_log`

## Призначення

Таблиця `audit_log` містить повну історію змін даних у системі.

Вона використовується для:

- аудиту;
- відновлення історії;
- пошуку помилок;
- аналізу змін;
- відстеження джерела даних.

На відміну від `timeline`, таблиця фіксує саме зміни записів у базі даних.

---

# Відповідальність

Таблиця відповідає за збереження історії змін усіх сутностей системи.

Жоден запис не редагується після створення.

---

# Поля

| Поле | Тип | NULL | Опис |
|------|-----|------|------|
| id | BIGSERIAL | ❌ | Первинний ключ |
| entity_type | VARCHAR(50) | ❌ | Тип сутності |
| entity_id | BIGINT | ❌ | Ідентифікатор сутності |
| action | VARCHAR(20) | ❌ | Тип операції |
| field_name | VARCHAR(100) | ✅ | Назва зміненого поля |
| old_value | TEXT | ✅ | Попереднє значення |
| new_value | TEXT | ✅ | Нове значення |
| changed_by | BIGINT | ✅ | users.id або NULL |
| source | VARCHAR(30) | ❌ | Джерело зміни |
| reason | TEXT | ✅ | Причина зміни |
| created_at | TIMESTAMP | ❌ | Час зміни |

---

---

# Правила оновлення

Записи Audit Log не змінюються.

Допускається лише створення нових записів.

---

# Видалення

Записи Audit Log не видаляються.

---

# Причини такої структури

Audit Log дозволяє відповісти на питання:

- Хто змінив дані?
- Коли це сталося?
- Що саме змінилося?
- Яке було попереднє значення?
- Звідки надійшла зміна?

---

# Майбутній розвиток

Можливе додавання:

- IP-адреси;
- User-Agent;
- Session ID;
- номера транзакції;
- групування декількох змін в одну операцію.
