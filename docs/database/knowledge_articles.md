# Таблиця `knowledge_articles`

## Призначення

Таблиця `knowledge_articles` містить структуровані статті бази знань спільноти.

Статті можуть створюватися:

- вручну;
- автоматично AI;
- на основі одного або декількох повідомлень;
- шляхом редагування існуючих матеріалів.

---

# Відповідальність

Таблиця відповідає за:

- зберігання статей;
- управління статусом;
- автора;
- історію публікації.

---

# Поля

| Поле | Тип | NULL | Опис |
|------|------|------|------|
| id | BIGSERIAL | ❌ | Первинний ключ |
| title | VARCHAR(255) | ❌ | Назва статті |
| slug | VARCHAR(255) | ❌ | URL-ідентифікатор |
| summary | TEXT | ✅ | Короткий опис |
| content | TEXT | ❌ | Основний текст |
| author_id | BIGINT | ✅ | Автор статті |
| source | VARCHAR(30) | ❌ | Джерело створення |
| status | VARCHAR(30) | ❌ | Статус |
| published_at | TIMESTAMP | ✅ | Дата публікації |
| created_at | TIMESTAMP | ❌ | Дата створення |
| updated_at | TIMESTAMP | ❌ | Дата оновлення |

# Правила оновлення

Стаття може необмежену кількість разів редагуватися.

Усі зміни повинні фіксуватися в `audit_log`.

---

# Причини такої структури

Knowledge Base є окремим модулем системи.

Статті не залежать від Telegram і можуть існувати навіть без початкових повідомлень.
