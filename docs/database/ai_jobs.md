# Таблиця `ai_jobs`

## Призначення

Таблиця `ai_jobs` містить чергу задач для модулів штучного інтелекту.

Вона дозволяє виконувати AI-аналіз асинхронно, контролювати стан задач, повторно запускати невдалі операції та уникати дублювання обробки.

---

# Відповідальність

Таблиця відповідає за:

- постановку AI-задач;
- контроль виконання;
- збереження статусу;
- облік помилок;
- повторний запуск задач.

---

# Поля

| Поле | Тип | NULL | Опис |
|------|------|------|------|
| id | BIGSERIAL | ❌ | Первинний ключ |
| job_type | VARCHAR(50) | ❌ | Тип задачі |
| entity_type | VARCHAR(50) | ❌ | Тип сутності |
| entity_id | BIGINT | ❌ | ID сутності |
| status | VARCHAR(30) | ❌ | Статус виконання |
| priority | SMALLINT | ❌ | Пріоритет |
| ai_model | VARCHAR(100) | ✅ | AI-модель |
| attempts | INTEGER | ❌ | Кількість спроб |
| error_message | TEXT | ✅ | Текст останньої помилки |
| started_at | TIMESTAMP | ✅ | Початок виконання |
| finished_at | TIMESTAMP | ✅ | Завершення виконання |
| created_at | TIMESTAMP | ❌ | Створення задачі |
| updated_at | TIMESTAMP | ❌ | Останнє оновлення |

---

---

# Правила оновлення

Після створення задачі допускається оновлення:

- статусу;
- кількості спроб;
- інформації про помилку;
- часу початку та завершення.

Тип задачі та сутність не змінюються.

---

# Причини такої структури

Використання окремої таблиці дозволяє:

- контролювати виконання AI;
- не виконувати однакові задачі повторно;
- масштабувати AI незалежно від Telegram Bot;
- будувати окремі AI-сервіси у майбутньому.

---

# Майбутній розвиток

Можливе додавання:

- часу очікування;
- часу виконання;
- окремих AI-черг;
- розподілу задач між декількома воркерами.
