# API

## Призначення

Цей документ описує всі способи взаємодії із платформою.

На першому етапі єдиним клієнтом є Telegram Bot.

У майбутньому документ буде доповнено REST API та Web API.

---

# Telegram Bot

Telegram Bot є основною точкою взаємодії користувача із системою.

Бот не містить бізнес-логіки.

Його завдання:

- приймати повідомлення;
- викликати відповідні сервіси;
- повертати результат користувачу.

# Callback Query

Інтерактивні елементи Telegram використовують Callback Data.

Формат:

```
<module>:<action>:<id>
```

Приклади:

```
event:open:15

profile:edit:42

knowledge:save:10
```

# REST API

На першому етапі не реалізовується.

Передбачається підтримка:

- авторизації;
- роботи із профілем;
- подій;
- Knowledge Base;
- аналітики.

---

# Принципи API

## Один запит — одна дія

Кожен endpoint або команда повинні виконувати лише одну логічну операцію.

---

## Ідемпотентність

Повторний запит не повинен призводити до дублювання даних.

---

## Єдиний формат відповідей

Будь-який сервіс повинен повертати передбачуваний результат.

---

# Версіонування

Після появи REST API використовується схема

```
/api/v1/
```

Нова версія API не повинна порушувати сумісність із попередньою.

---

# Авторизація

Telegram Bot використовує Telegram ID.

REST API у майбутньому використовуватиме окрему систему авторизації.

---

# Обмеження

Жоден клієнт не працює напряму з базою даних.

Усі запити проходять через Application Layer.
